Root Cause:
  In Laravel, routes are matched TOP-TO-BOTTOM in the order they're defined.
  When a request comes in, Laravel checks each route until it finds a match.
  If a wildcard route like {scholarship} is defined BEFORE an explicit route
  like eligibility-check, the wildcard will match first and the explicit route
  will never be reached.

Impact:
  This is a CRITICAL issue that breaks core functionality. Users cannot:
  âŒ Check scholarship eligibility
  âŒ Create community posts

Solution:
  Simple fix: Reorder routes - put specific/explicit routes BEFORE wildcard
  routes. This ensures requests are matched correctly.

Prevention:
  Always follow the pattern: Specific â†’ Semi-specific â†’ Wildcard
