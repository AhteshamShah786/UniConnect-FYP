KEY PRINCIPLE: ROUTE MATCHING IN LARAVEL
════════════════════════════════════════════════════════════════════════════════

Routes are checked in DEFINITION ORDER. First match wins!

    REQUEST comes in
         ↓
    Check Route 1 → Match? → YES: Use it / NO: Continue
         ↓
    Check Route 2 → Match? → YES: Use it / NO: Continue
         ↓
    Check Route 3 → Match? → YES: Use it / NO: Continue
         ↓
    ... (continue until match)
         ↓
    No routes matched → 404 ERROR

Therefore, you MUST define routes from MOST SPECIFIC to LEAST SPECIFIC:

    ┌────────────────────────────────────────┐
    │     MOST SPECIFIC (Exact)              │
    │  /resource/create                      │  ← Define these FIRST
    │  /resource/search                      │
    │  /resource/special-action              │
    ├────────────────────────────────────────┤
    │     SEMI-SPECIFIC (Pattern)            │
    │  /resource/{id}/edit                   │
    ├────────────────────────────────────────┤
    │     LEAST SPECIFIC (Wildcard)          │
    │  /resource/{id}                        │  ← Define these LAST
    └────────────────────────────────────────┘
